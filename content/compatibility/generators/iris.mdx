# 🌌 Iris

## Overview
Iris includes first-class Oraxen integration. Once both plugins are installed, Oraxen registers itself as an external data provider, so Iris can resolve your custom blocks directly inside pack configs.

<Callout type="info">
  No extra bridge, addon, or manual registration is required. Install both plugins and use the `oraxen:block_id` format in Iris configs.
</Callout>

## Block Format
Use the Oraxen namespace followed by your block ID.

```yml
oraxen:your_block_id
```

## Example
This example assumes you already created an Oraxen ore block, such as `amethyst_ore`.

<Steps>
### Open your dimension config
Go to `plugins/Iris/packs/YOUR_PACK_NAME/dimensions/YOUR_DIMENSION_NAME.json`.

For the default overworld pack, this is usually `plugins/Iris/packs/overworld/dimensions/overworld.json`.

### Find the ore deposit section
Locate the `deposits` list inside the dimension config.

```json
"deposits": [
  {
    "minHeight": 19,
    "maxPerChunk": 4,
    "maxHeight": 150,
    "minPerChunk": 1,
    "minSize": 25,
    "maxSize": 25,
    "palette": [{ "block": "granite" }],
    "varience": 2
  }
]
```

### Add your Oraxen ore
Create a new deposit entry and point the palette block to your Oraxen ID.

```json
{
  "minHeight": 2,
  "maxPerChunk": 2,
  "maxHeight": 30,
  "minPerChunk": 0,
  "minSize": 3,
  "maxSize": 6,
  "palette": [{ "block": "oraxen:amethyst_ore" }],
  "varience": 5
}
```

### Regenerate the world
Save the file, then generate fresh chunks or create a new world so Iris can place the new ore deposits.
</Steps>

## Notes
<Callout type="warning">
  World-generation changes only apply to newly generated chunks. Existing terrain will not retroactively gain your custom ores.
</Callout>

<Callout type="note">
  Iris pack paths can differ if you use a custom pack or dimension name, but the `oraxen:block_id` format stays the same.
</Callout>
