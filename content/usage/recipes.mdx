# đź“– Recipes

## Overview
Oraxen supports multiple custom recipe types and loads them from `.yml` files in the `plugins/Oraxen/recipes` folder. You can create recipes through the ingame builder, edit them manually, reload them without restarting the server, and disable vanilla or existing recipes by namespaced key.

### Supported Recipe Types
`shaped.yml` - Standard crafting recipes where ingredient position matters.<br />
`shapeless.yml` - Crafting recipes where ingredient order does not matter.<br />
`furnace.yml` - Furnace smelting recipes with time and experience.<br />
`blasting.yml` - Blast furnace recipes with time and experience.<br />
`campfire.yml` - Campfire cooking recipes with time and experience.<br />
`smoking.yml` - Smoker recipes with time and experience.<br />
`stonecutting.yml` - Stonecutter recipes with one input and one result.<br />
`disabled.yml` - Recipe keys that should be removed when Oraxen loads recipes.<br />

### Item Definitions
Recipe inputs and results support several item formats.

`oraxen_item` - Use an Oraxen item ID.<br />
`minecraft_type` - Use a vanilla material such as `DIAMOND` or `STICK`.<br />
`minecraft_item` - Use a fully serialized Bukkit `ItemStack`.<br />
`tag` - Use a vanilla item or block tag as an ingredient, such as `planks` or `minecraft:logs`. <Highlight color="#2E8B57">**Ingredients only**</Highlight><br />
`crucible_item` - Use a MythicCrucible item ID. <Highlight color="#2E8B57">**Optional compatibility**</Highlight><br />
`mmoitems_type` + `mmoitems_id` - Use an MMOItems item. <Highlight color="#2E8B57">**Optional compatibility**</Highlight><br />
`ecoitem_id` - Use an EcoItems item ID. <Highlight color="#2E8B57">**Optional compatibility**</Highlight><br />
`amount` - Set the stack size for a result, or the amount of one shapeless ingredient entry.<br />
`permission` - Require a permission to craft or unlock the recipe. <Highlight color="#2E8B57">**Optional**</Highlight><br />

<Callout type="info">
  Oraxen creates the recipe files automatically on first load. You can add entries manually or generate them ingame with the recipe builder.
</Callout>

## Recipe Formats

<details>
<summary><strong>đź§© Shaped</strong></summary>

Use `shape` and `ingredients` together. Each character in `shape` maps to one entry in `ingredients`, while `_` means an empty slot.

`shape` - Up to three crafting rows.<br />
`ingredients` - Character-to-item mapping.<br />
`result` - Crafted output item.<br />

```yaml
bedrock_pickaxe:
  shape:
    - AAA
    - BCB
    - _D_
  result:
    oraxen_item: bedrock_pickaxe
  ingredients:
    A:
      minecraft_type: DIAMOND_BLOCK
    B:
      minecraft_type: OBSIDIAN
    C:
      minecraft_type: NETHER_STAR
    D:
      minecraft_type: STICK
```
</details>

<details>
<summary><strong>đźŚ€ Shapeless</strong></summary>

Shapeless recipes ignore slot position. Each ingredient entry should include an `amount`.

`ingredients` - Arbitrary ingredient keys such as `A`, `B`, `C`.<br />
`amount` - How many times that ingredient must appear.<br />
`result` - Crafted output item.<br />

```yaml
obsidian_mix:
  result:
    oraxen_item: obsidian_ingot
  ingredients:
    A:
      minecraft_type: OBSIDIAN
      amount: 2
    B:
      tag: coals
      amount: 1
```
</details>

<details>
<summary><strong>đź”Ą Cooking Recipes</strong></summary>

`furnace.yml`, `blasting.yml`, `campfire.yml`, and `smoking.yml` all use the same structure.

`input` - The item that gets cooked.<br />
`result` - The cooked output item.<br />
`cookingTime` - Cooking duration in ticks.<br />
`experience` - Experience granted when the recipe completes.<br />

```yaml
amethyst:
  experience: 200
  cookingTime: 200
  result:
    oraxen_item: amethyst
  input:
    oraxen_item: amethyst_ore
```
</details>

<details>
<summary><strong>đźŞ¨ Stonecutting</strong></summary>

Stonecutting recipes use one `input` and one `result`.

```yaml
polished_obsidian:
  result:
    oraxen_item: polished_obsidian_tile
  input:
    minecraft_type: OBSIDIAN
```
</details>

## Ingame Builder

### Commands
You can create recipes ingame with the recipe builder commands listed on the [commands page](commands#recipes).

```yml
/oraxen recipes builder shaped
/oraxen recipes builder shapeless
/oraxen recipes builder furnace <cookingtime> <experience>
/oraxen recipes builder blasting <cookingtime> <experience>
/oraxen recipes builder campfire <cookingtime> <experience>
/oraxen recipes builder smoking <cookingtime> <experience>
/oraxen recipes builder stonecutting
/oraxen recipes save <name> [permission]
```

### Workflow
1. Open the builder for the recipe type you want.
2. Place the input ingredients in the builder UI.
3. Place the output item in the result slot.
4. Run `/oraxen recipes save <name>` to write the recipe to the matching `.yml` file.
5. Run `/oraxen reload recipes` or restart the server to register the new recipe.

<Callout type="note">
  Running `/oraxen recipes builder` with no type reopens your current builder if one is already active.
</Callout>

<Callout type="warning">
  Recipe builders are player-only commands. Cooking builders require both `cookingtime` and `experience` arguments when you create them.
</Callout>

## Recipe Showcase And Permissions

`permission` can be added to any recipe entry or passed when saving from the builder.<br />
Players without that permission cannot use the recipe.<br />

Crafting-table recipes can also appear in `/oraxen recipes show`, which opens Oraxen's showcase GUI for recipes the player is allowed to use.

<Callout type="info">
  The showcase GUI is built from Oraxen's whitelisted crafting recipes, so it applies to crafting-style recipes rather than furnace-like cooking recipes.
</Callout>

## Disabling Existing Recipes
You can disable vanilla or already-registered recipes by adding their namespaced keys to `disabled.yml`.

```yaml
disabled:
  - minecraft:diamond_sword
  - minecraft:iron_pickaxe
  - oraxen:custom_recipe
```

Oraxen attempts to remove each listed recipe whenever recipes load or reload.

<Callout type="warning">
  Invalid keys or recipes that do not exist will log a warning in the console.
</Callout>

## Notes

<Callout type="info">
  If you edit recipe files manually, use `/oraxen reload recipes` to apply the changes without a full server restart.
</Callout>

<Callout type="note">
  If `settings.yml` has recipe-book support enabled, Oraxen automatically discovers permitted recipes for players when they join.
</Callout>
