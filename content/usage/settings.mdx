# ŌÜÖ’ĖÅ Settings

## Overview
Oraxen is managed through the `/oraxen` root command. It also includes the `/o` and `/oxn` aliases. Most subcommands are intended for administrators, while a few are player-facing tools such as item browsing, emoji browsing, and HUD toggling.

## Settings
The examples below show the most important parts of `settings.yml`. Each section is collapsed into a `Details` block so you can quickly scan what a setting group does before opening the sample config.

<Details>
<summary id="debug"><strong>­¤Ü® `debug`</strong></summary>

Toggles debug mode, which will output detailed additional informational messages to the console. This option can also be toggled via the `/oraxen debug` command.

```yaml
debug: false
```

`debug` - Enables verbose console logging for Oraxen diagnostics. Use this when you need extra information while troubleshooting.<br />

</Details>

<Details>
<summary id="plugin"><strong>­¤¦® `Plugin`</strong></summary>

Controls Oraxen's global behavior. This includes config housekeeping, language selection, first-run messaging, repair-command behavior, Paper config tweaks for custom blocks, default asset/config generation, and which UI surfaces use MiniMessage formatting.

```yaml
Plugin:
  keep_this_up_to_date: true
  language: "english"
  introduction:
    enabled: true
    console_sent: false
  commands:
    repair:
      oraxen_durability_only: false
  auto_update_paper_config: true
  generation:
    default_assets: true
    default_configs: true
  formatting:
    inventory_titles: true
    titles: true
    subtitles: true
    action_bar: true
    anvil: true
    signs: true
    chat: true
    books: true
```

`keep_this_up_to_date` - Automatically updates the config when new settings are added by an update. Oraxen may delete old settings when this is enabled.<br />
`language` - Selects the language file Oraxen uses for messages. Valid options include `czech`, `english`, `french`, `german`, `jpn_JP`, `korean`, `ru-RU`, `zh-CN`, and `zh-TW`.<br />
`introduction.enabled` - Shows the first-run welcome message. Set this to `false` to suppress it entirely.<br />
`introduction.console_sent` - Internal flag set after the welcome message has been shown in console.<br />
`commands.repair.oraxen_durability_only` - Makes `/oraxen repair` skip vanilla item durability if enabled.<br />
`auto_update_paper_config` - Updates `paper-global.yml` to disable block updates for custom blocks.<br />
`generation.default_assets` - Generates the default resource-pack assets that ship with Oraxen.<br />
`generation.default_configs` - Generates the default config files that Oraxen provides.<br />
`formatting.inventory_titles` - Formats inventory titles with MiniMessage.<br />
`formatting.titles` - Formats title packets with MiniMessage.<br />
`formatting.subtitles` - Formats subtitle packets with MiniMessage.<br />
`formatting.action_bar` - Formats action bar messages with MiniMessage.<br />
`formatting.anvil` - Formats anvil result text sent by plugins.<br />
`formatting.signs` - Formats sign text when placed or edited.<br />
`formatting.chat` - Formats chat messages and glyphs in chat. Disable this if a chat plugin conflicts with it.<br />
`formatting.books` - Formats glyphs shown in written books.<br />

<Callout type="info">
	The `formatting` section uses MiniMessage. Legacy color formats such as `┬¦c` or `&c` are not supported here.
</Callout>

</Details>

<summary id="worldedit"><strong>­¤¦▒ `WorldEdit`</strong></summary>

Experimental clipboard support for note blocks, string blocks, and furniture when copying and pasting with WorldEdit.

```yaml
WorldEdit:
  noteblock_mechanic: false
  stringblock_mechanic: false
  furniture_mechanic: false
```

`noteblock_mechanic` - Enables WorldEdit support for copying and pasting note blocks.<br />
`stringblock_mechanic` - Enables WorldEdit support for copying and pasting string blocks. This works, but can be buggy with some string blocks, especially ones using the `Tall` property.<br />
`furniture_mechanic` - Enables WorldEdit support for copying and pasting furniture. This only handles clipboard operations such as cut, copy, and paste.<br />

<Callout type="warning">
	These WorldEdit integrations are experimental and should be used with caution.
</Callout>

</Details>

<Details>
<summary id="configstools"><strong>­¤øĀ’ĖÅ `ConfigsTools`</strong></summary>

Controls config migration helpers, model-data skipping, glyph-code generation, and the fallback item shown when a config entry cannot be resolved.

```yaml
ConfigsTools:
  skipped_model_data_numbers: []
  disable_automatic_model_data: false
  disable_automatic_glyph_code: false
  enable_configs_updater: true
  error_item:
    material: PODZOL
    excludeFromInventory: false
    injectID: false
```

`skipped_model_data_numbers` - Reserves specific CustomModelData values so Oraxen will not assign them automatically. This list accepts singular numbers and ranges such as `1-4`.<br />
`disable_automatic_model_data` - Stops Oraxen from assigning model data values on its own.<br />
`disable_automatic_glyph_code` - Stops Oraxen from auto-generating glyph codes.<br />
`enable_configs_updater` - Enables the config updater that migrates older configs to newer formats.<br />
`error_item.material` - Material used for the fallback item shown when a config entry fails.<br />
`error_item.excludeFromInventory` - Hides the fallback item from inventories when enabled.<br />
`error_item.injectID` - Injects the invalid entry's ID into the fallback item for easier debugging.<br />

</Details>

<Details>
<summary id="glyphs"><strong>Ō£” `Glyphs`</strong></summary>

Configures the glyph backend, whether emojis are hidden behind permissions, how Unicode completions behave, and the hover text shown when a glyph is displayed in chat.

```yaml
Glyphs:
  glyph_handler: VANILLA
  emoji_list_permission_only: true
  unicode_completions: true
  chat_hover_text: "<glyph_placeholder>"
```

`glyph_handler` - Chooses the backend Oraxen uses to render glyphs. Valid handlers are `VANILLA` and `NMS`.<br />
`emoji_list_permission_only` - Shows only emojis the player has permission to use.<br />
`unicode_completions` - Makes glyph tab completion show Unicode values instead of glyph placeholders.<br />
`chat_hover_text` - Sets the hover text shown for glyphs in chat.<br />

</Details>

<Details>
<summary id="texteffects"><strong>Ō£© `TextEffects`</strong></summary>

Enables shader-driven animated text effects such as rainbow, wave, and shake, and controls which shader template and encoding strategy Oraxen uses when rendering them. Effect definitions and GLSL snippets live in `text_effects.yml`.

```yaml
TextEffects:
  enabled: true
  shader:
    template: auto
    encoding: effect_font
```

`enabled` - Master toggle for all shader-based text effects.<br />
`shader.template` - Chooses which shader template Oraxen uses for text effects. Valid options are `auto`, `effects_only`, and `animated_only`.<br />
`shader.encoding` - Chooses how text effect data is encoded for shaders. The default `effect_font` strategy uses dedicated effect font glyphs.<br />

<Callout type="info">
	Text effects are compatible with Minecraft 1.21.11+.
</Callout>

</Details>

<Details>
<summary id="chat"><strong>­¤Æ¼ `Chat`</strong></summary>

Controls how glyphs are rendered in chat. `MODERN` is the default for current servers, while `LEGACY` is the fallback when another chat pipeline interferes with glyph rendering.

```yaml
Chat:
  chat_handler: MODERN
```

`chat_handler` - Chooses the chat rendering mode used for glyphs in chat. Valid options are `MODERN` and `LEGACY`. Use `LEGACY` only if `MODERN` causes display issues.<br />

</Details>

<Details>
<summary id="customarmor"><strong>­¤øĪ’ĖÅ `CustomArmor`</strong></summary>

Controls which custom armor system Oraxen uses, whether leather repair is allowed, and how component, trim, or shader-based armor assets are generated.

```yaml
CustomArmor:
  type: COMPONENT
  auto_select_type: true
  disable_leather_repair: true
  component_settings:
    auto_assign_component: true
  trims_settings:
    material_replacement: CHAINMAIL
    auto_assign_settings: true
  shader_settings:
    type: FANCY
    armor_resolution: 16
    animated_armor_framerate: 24
    generate_shader_compatible_armor: true
    generate_armor_shader_files: true
    generate_custom_armor_textures: true
```

`type` - Selects the custom armor system Oraxen should use. Valid types are `COMPONENT`, `SHADER`, `TRIMS`, and `NONE`.<br />
`auto_select_type` - Lets Oraxen replace incompatible armor types with the best type for the detected server version.<br />
`disable_leather_repair` - Prevents custom armor from being repaired with leather. Custom armor can only be repaired with copies of the same custom armor.<br />
`component_settings.auto_assign_component` - Automatically adds the equippable-model component unless you set it manually.<br />
`trims_settings.material_replacement` - Chooses the vanilla material used to represent trim-based armor. Valid materials are `CHAINMAIL`, `IRON`, `GOLD`, `DIAMOND`, `NETHERITE`, and `LEATHER`.<br />
`trims_settings.auto_assign_settings` - Automatically applies trim and item-flag settings for custom armor.<br />
`shader_settings.type` - Chooses the shader armor style. Valid types are `FANCY` and `LESS_FANCY`.<br />
`shader_settings.armor_resolution` - Sets the source armor texture resolution used for shader generation. For example, `64x32` maps to `16`, and `128x64` maps to `32`.<br />
`shader_settings.animated_armor_framerate` - Controls the framerate used for animated armor textures.<br />
`shader_settings.generate_shader_compatible_armor` - Generates armor output compatible with the shader armor pipeline.<br />
`shader_settings.generate_armor_shader_files` - Generates the shader files needed for custom armor.<br />
`shader_settings.generate_custom_armor_textures` - Generates the custom armor textures themselves.<br />

</Details>

<summary id="customblocks"><strong>­¤¦▒ `CustomBlocks`</strong></summary>

Handles block correction and note-block compatibility behavior for custom blocks.

```yaml
CustomBlocks:
  block_correction: NMS
  use_legacy_noteblocks: true
```

`block_correction` - Chooses how Oraxen corrects custom blocks after placement or load. Valid types are `NMS` and `LEGACY`.<br />
`use_legacy_noteblocks` - Keeps legacy note-block behavior enabled for compatibility. This setting is relevant for future changes; for now, leave it enabled unless you know you need otherwise.<br />

<Details>
<summary id="itemupdater"><strong>­¤öü `ItemUpdater`</strong></summary>

Keeps player inventories, loaded block inventories, and loaded entity-held items synced to the latest Oraxen item definitions.

```yaml
ItemUpdater:
  update_items: true
  update_tile_entity_contents: true
  update_entity_contents: true
  override_renamed_items: false
  override_item_lore: false
  update_items_on_reload: true
```

`update_items` - Updates player inventories to the current item config when they join.<br />
`update_tile_entity_contents` - Updates items in loaded block inventories such as chests and furnaces.<br />
`update_entity_contents` - Updates items held by loaded entities such as item frames, mobs, and ender chests.<br />
`override_renamed_items` - Replaces items even if they were renamed by a player.<br />
`override_item_lore` - Forces item lore to match the Oraxen item definition.<br />
`update_items_on_reload` - Re-applies item updates to online players after `/oraxen reload items` or `/oraxen reload all`.<br />

</Details>

<Details>
<summary id="furnitureupdater"><strong>­¤¬æ `FurnitureUpdater`</strong></summary>

Keeps placed furniture synced when players join, reload items, or load entities and chunks, and can try to repair broken furniture.

```yaml
FurnitureUpdater:
  update_furniture: true
  update_on_reload: false
  update_on_load: false
  experimental_furniture_type_update: false
  experimental_fix_broken_furniture: false
```

`update_furniture` - Keeps placed furniture synced with the current config.<br />
`update_on_reload` - Updates furniture after `/oraxen reload items` or `/oraxen reload all`.<br />
`update_on_load` - Updates the furniture item whenever an entity is loaded.<br />
`experimental_furniture_type_update` - Enables experimental furniture type migration.<br />
`experimental_fix_broken_furniture` - Tries to repair furniture that is not placed or broken correctly.<br />

</Details>

<summary id="pack"><strong>­¤ō” `Pack`</strong></summary>

Controls the resource-pack pipeline: generation, importing external pack content, uploading, dispatching to players, and handling pack-status feedback.

### Generation

```yaml
Pack:
  generation:
    generate: true
    disable_mcmeta_generation: false
    multi_version_packs: false
    excluded_file_extensions: []
    appearance:
      item_properties: true
      model_data_ids: false
      model_data_float: false
      generate_predicates: false
    verify_pack_files: true
    fix_force_unicode_glyphs: true
    texture_slicer: true
    atlas:
      exclude_malformed_from_atlas: true
      generate: true
      type: "SPRITE"
    auto_generated_models_follow_texture_path: false
    compression: BEST_COMPRESSION
    obfuscation:
      type: NONE
    protection: true
    comment: "The content of this texture pack\n..."
```

`generation.generate` - Enables or disables all resource-pack generation.<br />
`generation.disable_mcmeta_generation` - Prevents Oraxen from creating or modifying `pack.mcmeta`. The file at `plugins/Oraxen/pack/pack.mcmeta` is included exactly as provided, and automatic overlay entries are disabled.<br />
`generation.multi_version_packs` - Generates separate packs with correct `pack_format` values for different client versions. This is intended for servers using ViaVersion or ProtocolSupport with upload types that can serve multiple files.<br />
`generation.excluded_file_extensions` - Lists file extensions that should not be included in the final zipped resource pack.<br />
`generation.appearance.item_properties` - Uses the `minecraft:item_model` component with Oraxen item IDs and generates `assets/oraxen/items/<item_id>.json` for Minecraft 1.21.4+.<br />
`generation.appearance.model_data_ids` - Uses `custom_model_data.strings[0]` with stable string identifiers such as `oraxen:<item_id>` and generates `minecraft:select` item definitions.<br />
`generation.appearance.model_data_float` - Uses `custom_model_data.floats[0]` with `minecraft:range_dispatch` and also sets integer CustomModelData for plugin compatibility.<br />
`generation.appearance.generate_predicates` - Generates legacy predicate override files on 1.21.4+. This is not normally needed unless an external tool requires legacy JSON files. Pre-1.21.4 servers always generate predicates.<br />
`generation.verify_pack_files` - Checks textures and models for valid resource-pack formatting.<br />
`generation.fix_force_unicode_glyphs` - Fixes glyphs not showing when Force Unicode is enabled.<br />
`generation.texture_slicer` - Slices default-texture overrides into the newer 1.20.2 format.<br />
`generation.atlas.exclude_malformed_from_atlas` - Excludes malformed textures from the atlas instead of including invalid entries.<br />
`generation.atlas.generate` - Generates atlas files for the resource pack.<br />
`generation.atlas.type` - Chooses the atlas mode. Valid options are `SPRITE` and `DIRECTORY`.<br />
`generation.auto_generated_models_follow_texture_path` - Makes auto-generated models follow the source texture path.<br />
`generation.compression` - Chooses the ZIP compression level used for the generated pack. This follows Java's `Deflater` constants.<br />
`generation.obfuscation.type` - Chooses the pack obfuscation mode. `NONE` keeps keys readable, `SIMPLE` renames pack-internal model, texture, and sound paths while preserving namespaces, and `FULL` also replaces namespaces used by renamed resources.<br />
`generation.protection` - Applies protection methods intended to make the pack difficult to extract with common archive tools.<br />
`generation.comment` - Adds a comment to the generated pack archive.<br />

<Callout type="warning">
	Do not try to extract a pack after enabling `generation.protection`; protected archives can behave unexpectedly in normal archive tools.
</Callout>

<Callout type="info">
	`generation.appearance.model_data_ids` and `generation.appearance.model_data_float` cannot be combined because both write to the same item definition files.
</Callout>

### Import

```yaml
Pack:
  import:
    merge_duplicate_fonts: true
    merge_duplicates: true
    remove_core_shaders_from_imported_packs: false
    retain_custom_model_data: true
    merge_item_base_models: false
```

`import.merge_duplicate_fonts` - Merges duplicate font files, most often `default.json`, into the final pack. Original files are not deleted, but duplicates are excluded in favor of the merged file.<br />
`import.merge_duplicates` - Merges duplicate imported resources when possible.<br />
`import.remove_core_shaders_from_imported_packs` - Removes imported Minecraft core shader files from `pack/assets` and uploaded ZIP merges. This is useful when external packs override Oraxen-generated text or scoreboard shaders.<br />
`import.retain_custom_model_data` - Keeps CustomModelData values from imported files. Only enable this when another plugin requires those exact values, because it can conflict with other Oraxen items.<br />
`import.merge_item_base_models` - Attempts to merge item base model JSON files such as `paper.json` from imported packs instead of generating item configs from them.<br />

### Upload

```yaml
Pack:
  upload:
    enabled: true
    type: polymath
    polymath:
      server: atlas.oraxen.com
      secret: "oraxen"
    lobfile:
      api-key: ""
      pack-name: Oraxen
    self-host:
      host: "0.0.0.0"
      port: 8080
      domain: "localhost:8080"
```

`upload.enabled` - Enables uploading the generated resource pack to the selected host.<br />
`upload.type` - Chooses the upload backend. Valid options are `lobfile`, `polymath`, `self-host`, and `external`.<br />
`upload.polymath.server` - Sets the Polymath server used for resource-pack uploads. You can use the public Oraxen Atlas endpoint or your own Polymath instance.<br />
`upload.polymath.secret` - Sets the Polymath upload secret. Change this when hosting your own Polymath instance to limit upload access.<br />
`upload.lobfile.api-key` - Sets the Lobfile API key used for uploads.<br />
`upload.lobfile.pack-name` - Sets the Lobfile pack name. When multi-version packs are enabled, Oraxen uses `<pack-name>_<version>`.<br />
`upload.self-host.host` - Sets the IP address the self-hosted HTTP server binds to. `0.0.0.0` listens on all interfaces.<br />
`upload.self-host.port` - Sets the port the self-hosted HTTP server listens on.<br />
`upload.self-host.domain` - Sets the domain or IP and port players use to download the self-hosted pack.<br />

### Dispatch

```yaml
Pack:
  dispatch:
    send_pre_join: false
    send_on_join: true
    send_on_reload: true
    delay: -1
    mandatory: true
    disable_movement_on_load: true
    disable_damage_on_load: true
    prompt: "<#fa4943>Accept the pack to enjoy a full <b><gradient:#9055FF:#13E2DA>Oraxen</b><#fa4943> experience"
    layer: ""
    join_message:
      enabled: false
      delay: -1
```

`dispatch.send_pre_join` - Sends the pack before join on supported Paper versions. This takes priority over `send_on_join` when supported and requires explicit opt-in.<br />
`dispatch.send_on_join` - Sends the pack after the player fully joins.<br />
`dispatch.send_on_reload` - Resends the pack when Oraxen reloads.<br />
`dispatch.delay` - Delays pack sending by the configured number of ticks. `-1` uses Oraxen's default timing.<br />
`dispatch.mandatory` - Makes the pack required instead of optional.<br />
`dispatch.disable_movement_on_load` - Freezes player movement while the pack loads.<br />
`dispatch.disable_damage_on_load` - Prevents damage while the pack loads.<br />
`dispatch.prompt` - Sets the resource-pack prompt shown to players.<br />
`dispatch.layer` - Sets a resource-pack layer name for BungeeCord or Velocity compatibility. Any non-empty value makes Oraxen use `addResourcePack` instead of `setResourcePack`, allowing multiple packs to coexist on Minecraft 1.20.3+.<br />
`dispatch.join_message.enabled` - Sends a join message related to the resource pack when enabled.<br />
`dispatch.join_message.delay` - Delays the join message by the configured number of ticks. `-1` uses Oraxen's default timing.<br />

### Receive

```yaml
Pack:
  receive:
    enabled: true
    loaded:
      actions:
        sound:
          enabled: true
          type: minecraft:welcome
          volume: 1.0
          pitch: 1.0
        delay: 5
        message:
          enabled: false
          type: ACTION_BAR
          content: "<#55ffa4><bold>ResourcePack loaded"
        commands:
          console: []
          player: []
          opped_player: []
    accepted:
      actions:
        message:
          enabled: false
          type: ACTION_BAR
          content: "<#55ffa4><bold>ResourcePack accepted"
        commands:
          console: []
          player: []
          opped_player: []
    denied:
      actions:
        message:
          enabled: false
          type: CHAT
          content: "<red>You refused the ResourcePack..."
        commands:
          console: []
          player: []
          opped_player: []
    failed_download:
      actions:
        message:
          enabled: false
          type: CHAT
          content: "<red>You failed to download the ResourcePack..."
        commands:
          console: []
          player: []
          opped_player: []
    failed_reload:
      actions:
        commands:
          console: []
          player: []
          opped_player: []
    downloaded:
      actions:
        commands:
          console: []
          player: []
          opped_player: []
    invalid_url:
      actions:
        commands:
          console: []
          player: []
          opped_player: []
    discarded:
      actions:
        commands:
          console: []
          player: []
          opped_player: []
```

`receive.enabled` - Enables pack status feedback handling.<br />
`receive.loaded.actions.sound.enabled` - Plays a sound after the pack is loaded when enabled.<br />
`receive.loaded.actions.sound.type` - Sets the sound played after the pack is loaded.<br />
`receive.loaded.actions.sound.volume` - Sets the loaded sound volume.<br />
`receive.loaded.actions.sound.pitch` - Sets the loaded sound pitch.<br />
`receive.loaded.actions.delay` - Delays loaded actions by the configured number of ticks.<br />
`receive.<status>.actions.message.enabled` - Sends a message for the given pack status when enabled.<br />
`receive.<status>.actions.message.type` - Chooses how the message is delivered. Valid options are `KICK`, `CHAT`, `ACTION_BAR`, and `TITLE`.<br />
`receive.<status>.actions.message.content` - Sets the message content. Click and hover elements are only available with `CHAT` messages.<br />
`receive.<status>.actions.commands.console` - Runs commands as console for the given pack status.<br />
`receive.<status>.actions.commands.player` - Runs commands as the player for the given pack status.<br />
`receive.<status>.actions.commands.opped_player` - Runs commands as a temporarily opped player for the given pack status.<br />

Supported receive statuses are `loaded`, `accepted`, `denied`, `failed_download`, `failed_reload`, `downloaded`, `invalid_url`, and `discarded`. The `failed_reload`, `downloaded`, `invalid_url`, and `discarded` statuses are available on Minecraft 1.20.3+.

</Details>

<summary id="misc"><strong>ŌÜÖ’ĖÅ `Misc`</strong></summary>

Contains the smaller global toggles that do not fit elsewhere, including recipes, scoreboard and tablist visuals, armor-equip bypasses, and vanilla item display overrides.

```yaml
Misc:
  reset_recipes: true
  add_recipes_to_book: true
  hide_scoreboard_numbers: false
  hide_scoreboard_background: false
  hide_tablist_background: false
  armor_equip_event_bypass:
    - FURNACE
    - CHEST
    - TRAPPED_CHEST
  shield_display: '{"thirdperson_righthand":{"rotation":[0,90,0],"translation":[10,6,-4],"scale":[1,1,1]}}'
  bow_display: '{"thirdperson_righthand":{"rotation":[-80,260,-40],"translation":[-1,-2,2.5],"scale":[0.9,0.9,0.9]}}'
  crossbow_display: '{"thirdperson_righthand":{"rotation":[-90,0,-60],"translation":[2,0.1,-3],"scale":[0.9,0.9,0.9]}}'
```

`reset_recipes` - Resets Oraxen recipes on reload. This can conflict with other plugins, but makes recipe testing easier.<br />
`add_recipes_to_book` - Adds unlocked Oraxen recipes to the player recipe book.<br />
`hide_scoreboard_numbers` - Hides scoreboard numbers on the sidebar.<br />
`hide_scoreboard_background` - Hides the scoreboard background.<br />
`hide_tablist_background` - Hides the tablist background on 1.21+.<br />
`armor_equip_event_bypass` - Lists blocks that should not trigger armor equip behavior when right-clicked. This should include blocks with inventories and other blocks that should not equip armor on interaction.<br />
`shield_display` - Sets the vanilla shield display transform used when Oraxen needs to add custom shields.<br />
`bow_display` - Sets the vanilla bow display transform used when Oraxen needs to add custom bows.<br />
`crossbow_display` - Sets the vanilla crossbow display transform used when Oraxen needs to add custom crossbows.<br />

<Callout type="warning">
	Due to Mojang core shader changes, hiding the tablist or scoreboard background may not work on 1.21.8+.
</Callout>
</Details>

<summary id="oraxen_inventory"><strong>­¤Śé’ĖÅ `oraxen_inventory`</strong></summary>

Configures the built-in browser GUI that players open with Oraxen commands. It controls the menu title, pagination, and the category buttons shown in the item browser.

```yaml
oraxen_inventory:
  main_menu_title: "<shift:-18><glyph:menu_items><shift:-193>"
  menu_rows: 6
  menu_size: 45
  #exit_icon: exit_icon
  #next_page_icon: next_page_icon
  #previous_page_icon: previous_page_icon
  menu_layout:
    armors:
      slot: 1
      icon: emerald_chestplate
      displayname: "<green>Armors"
      title: "<main_menu_title><#362753><glyph:menu_items_overlay:colorable>"
```

`main_menu_title` - Sets the title used for the main browser GUI.<br />
`menu_rows` - Sets how many rows the GUI uses.<br />
`menu_size` - Sets how many item slots are shown per page.<br />
`exit_icon` - Optional icon used for the exit button.<br />
`next_page_icon` - Optional icon used for the next-page button.<br />
`previous_page_icon` - Optional icon used for the previous-page button.<br />
