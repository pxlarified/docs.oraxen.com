# 🖌️ Appearance

## Overview
Item appearance is controlled by the `Pack` section in each item config and by the appearance systems enabled in `settings.yml`.
Oraxen generates the required resource pack files (models, textures, and item definitions) based on those settings.

<Callout type="info">
    On `1.21.4+`, Oraxen can combine multiple appearance systems: the modern `minecraft:item_model` component and CustomModelData dispatchers.
    On older versions, it always falls back to legacy predicate overrides with integer CustomModelData.
</Callout>

## Configuration
Appearance is configured per item in `.yml` files inside `plugins/Oraxen/items`, while the model/texture assets live in `plugins/Oraxen/pack`.

## Appearance Systems (settings.yml)
The appearance systems are configured in `plugins/Oraxen/settings.yml` under `Pack.generation.appearance`.
This controls what resource pack files are generated by Oraxen.

```yaml
Pack:
  generation:
    appearance:
      item_properties: true
      model_data_ids: false
      model_data_float: false
      generate_predicates: false
```

<Details>
    <summary><h4>🧩 `item_properties`</h4></summary>
Uses the `minecraft:item_model` component (1.21.4+) with keys like `oraxen:<item_id>`.
Oraxen generates item definition files at `assets/oraxen/items/<item_id>.json` that point to your model.

- **Type -** `boolean`
- **Default -** `true` on `1.21.4+`
</Details>

<Details>
    <summary><h4>🏷️ `model_data_ids`</h4></summary>
Uses `custom_model_data.strings[0] = "oraxen:<item_id>"` and generates `assets/minecraft/items/<material>.json`
with `minecraft:select` dispatchers.

- **Type -** `boolean`
- **Default -** `false`
</Details>

<Details>
    <summary><h4>🔢 `model_data_float`</h4></summary>
Uses `custom_model_data.floats[0] = <custom_model_data>` and generates `assets/minecraft/items/<material>.json`
with `minecraft:range_dispatch`. Oraxen also sets integer CustomModelData on the item for compatibility.

- **Type -** `boolean`
- **Default -** `false`
</Details>

<Details>
    <summary><h4>🧱 `generate_predicates`</h4></summary>
Generates legacy predicate overrides in `assets/minecraft/models/item/*.json`. Not required on `1.21.4+`,
but useful when external tools still read legacy predicate models.

- **Type -** `boolean`
- **Default -** `false`
</Details>

<Callout type="info">
    On `1.21.4+`, you can enable both `model_data_ids` and `model_data_float` to generate both dispatch modes
    in vanilla item definitions for compatibility. Oraxen will still set both string and float data on items.
</Callout>

## Pack Properties

<Details>
    <summary><h4>🖼️ `model`</h4></summary>
Path to an existing model JSON (relative to `pack/models/`, without `.json`).

```yaml
my_item:
  Pack:
    model: weapons/swords/ice_sword
```

- **Type -** `string`
</Details>

<Details>
    <summary><h4>🧾 `gui_model`</h4></summary>
Model used in inventory/GUI contexts. When set, `model` is used for held/equipped contexts.

```yaml
my_item:
  Pack:
    model: weapons/swords/ice_sword_3d
    gui_model: weapons/swords/ice_sword_icon
```

- **Type -** `string`
- **Minecraft version -** `1.21.2+`
</Details>

<Details>
    <summary><h4>⚙️ `generate_model`</h4></summary>
Automatically generates a model JSON from `textures`. Defaults to `true` when `model` is not set.

```yaml
my_item:
  Pack:
    generate_model: true
    parent_model: item/generated
    textures:
      - weapons/swords/ice_sword
```

- **Type -** `boolean`
</Details>

<Details>
    <summary><h4>🎨 `textures`</h4></summary>
Texture layers for generated models (relative to `pack/textures/`, without `.png`).
Accepts either an array (layer0, layer1, ...) or a named map.

```yaml
my_item:
  Pack:
    textures:
      - weapons/swords/ice_sword
```

```yaml
my_item:
  Pack:
    textures:
      layer0: weapons/swords/ice_sword
      particle: weapons/swords/ice_sword
```

- **Type -** `array` or `object`
</Details>

<Details>
    <summary><h4>📦 `parent_model`</h4></summary>
Parent model used when `generate_model` is enabled.

```yaml
my_item:
  Pack:
    parent_model: item/handheld
```

- **Type -** `string`
- **Default -** `item/generated`
</Details>

<Details>
    <summary><h4>🧭 `generated_model_path`</h4></summary>
Custom output path for generated model files. Supports namespaced paths like `oraxen:models/weapons`.

```yaml
my_item:
  Pack:
    generated_model_path: oraxen:models/weapons
```

- **Type -** `string`
</Details>

<Details>
    <summary><h4>🔢 `custom_model_data`</h4></summary>
Sets a specific CustomModelData value (auto-generated if not set). Used by legacy predicates and
`model_data_float` dispatching.

```yaml
my_item:
  Pack:
    custom_model_data: 120001
```

- **Type -** `integer`
</Details>

<Details>
    <summary><h4>🧩 `models`</h4></summary>
Additional model definitions registered as `oraxen:<item_id>/<key>`. Useful for mechanics that swap models.

```yaml
my_item:
  Pack:
    models:
      active: weapons/swords/ice_sword_active
      cracked: weapons/swords/ice_sword_cracked
```

- **Type -** `object`
</Details>

<Details>
    <summary><h4>🛡️ `blocking_model` / `blocking_texture`</h4></summary>
Shield blocking model or texture. If you provide a texture, Oraxen generates a model automatically.

```yaml
my_shield:
  Pack:
    blocking_texture: weapons/shields/ice_shield_blocking
```

- **Type -** `string`
</Details>

<Details>
    <summary><h4>🏹 `pulling_models` / `pulling_textures`</h4></summary>
Bow pulling stage models or textures. Provide multiple entries (from early to late pull stages).

```yaml
my_bow:
  Pack:
    pulling_models:
      - weapons/bows/ice_bow_pull_0
      - weapons/bows/ice_bow_pull_1
      - weapons/bows/ice_bow_pull_2
```

- **Type -** `array`
</Details>

<Details>
    <summary><h4>⚡ `charged_model` / `charged_texture`</h4></summary>
Crossbow charged model or texture.

```yaml
my_crossbow:
  Pack:
    charged_model: weapons/crossbows/ice_crossbow_charged
```

- **Type -** `string`
</Details>

<Details>
    <summary><h4>🎆 `firework_model` / `firework_texture`</h4></summary>
Crossbow-with-firework model or texture.

```yaml
my_crossbow:
  Pack:
    firework_texture: weapons/crossbows/ice_crossbow_firework
```

- **Type -** `string`
</Details>

<Details>
    <summary><h4>🎣 `cast_model` / `cast_texture`</h4></summary>
Fishing rod cast model or texture.

```yaml
my_rod:
  Pack:
    cast_model: tools/rods/ice_rod_cast
```

- **Type -** `string`
</Details>

<Details>
    <summary><h4>🩹 `damaged_models` / `damaged_textures`</h4></summary>
Durability-based model or texture stages, from least to most damaged.

```yaml
my_item:
  Pack:
    damaged_textures:
      - tools/picks/ice_pick_1
      - tools/picks/ice_pick_2
      - tools/picks/ice_pick_3
```

- **Type -** `array`
</Details>

<Details>
    <summary><h4>🚫 `exclude_from_predicates`</h4></summary>
Exclude this item from CustomModelData predicate generation.

```yaml
my_item:
  Pack:
    exclude_from_predicates: true
```

- **Type -** `boolean`
- **Minecraft version -** `1.21.4+`
</Details>

<Details>
    <summary><h4>🚫 `exclude_from_item_model`</h4></summary>
Exclude this item from the `minecraft:item_model` component.

```yaml
my_item:
  Pack:
    exclude_from_item_model: true
```

- **Type -** `boolean`
- **Minecraft version -** `1.21.4+`
</Details>

<Details>
    <summary><h4>📏 `oversized_in_gui`</h4></summary>
Allows oversized rendering in GUI contexts for item definitions.

```yaml
my_item:
  Pack:
    oversized_in_gui: true
```

- **Type -** `boolean`
- **Minecraft version -** `1.21.4+`
</Details>

<Details>
    <summary><h4>🤚 `hand_animation_on_swap`</h4></summary>
Controls whether the hand animation plays when swapping to this item.

```yaml
my_item:
  Pack:
    hand_animation_on_swap: false
```

- **Type -** `boolean`
- **Default -** `true`
- **Minecraft version -** `1.21.4+`
</Details>

<Details>
    <summary><h4>📐 `swap_animation_scale`</h4></summary>
Scales the item swap animation.

```yaml
my_item:
  Pack:
    swap_animation_scale: 0.75
```

- **Type -** `number`
- **Default -** `1.0`
- **Minecraft version -** `1.21.4+`
</Details>

## Example

```yaml
ice_sword:
  material: IRON_SWORD
  itemname: "<gradient:#80d9ff:#1c7fd1>Ice Sword"
  Pack:
    model: weapons/swords/ice_sword
    gui_model: weapons/swords/ice_sword_icon
    generate_model: false
    custom_model_data: 120001
    models:
      active: weapons/swords/ice_sword_active
    damaged_textures:
      - weapons/swords/ice_sword_cracked_1
      - weapons/swords/ice_sword_cracked_2
```

## Notes
- Pack model paths are relative to `plugins/Oraxen/pack/models` and omit the `.json` extension.
- Pack texture paths are relative to `plugins/Oraxen/pack/textures` and omit the `.png` extension.
- `custom_model_data` is optional; Oraxen will auto-generate it unless automatic model data is disabled.
- This page focuses on appearance; properties and components are documented separately.
