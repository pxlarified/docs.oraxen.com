# 🧩 Components

## Overview
Components define modern vanilla item behavior in Oraxen, such as food values, durability, equipping, stack size, cooldowns, and custom tooltip or model settings.

<Callout>
    Didn't found the component you need? Oraxen supports all vanilla (and custom) components. See more <Highlight color="#FFA500" to="#generic">**here**</Highlight>.
</Callout>

## Configuration
Components are configured per item in `.yml` files inside the `plugins/Oraxen/items` directory.

## Components

<Details>
<summary id="max_stack_size"><strong>📦 `max_stack_size`</strong></summary>
Sets the maximum stack size of the item.<br />

```yaml
myitem:
  components:
    max_stack_size: 64
```

**Range**<br />
`1` to `99`

**Minecraft version**<br />
`1.20.5+`
</Details>

<Details>
<summary id="fire_resistant"><strong>🔥 `fire_resistant`</strong></summary>
Makes the item immune to fire and lava damage when dropped.<br />

```yaml
myitem:
  components:
    fire_resistant: true
```

**Minecraft version**<br />
`1.20.5+`
</Details>

<Details>
<summary id="hide_tooltip"><strong>🙈 `hide_tooltip`</strong></summary>
Hides the item's tooltip entirely.<br />

```yaml
myitem:
  components:
    hide_tooltip: true
```

**Minecraft version**<br />
`1.20.5+`
</Details>

<Details>
<summary id="durability"><strong>🔨 `durability`</strong></summary>
Defines custom durability behavior for the item.<br />

```yaml
myitem:
  components:
    durability:
      value: 250
      damage_block_break: true
      damage_entity_hit: true
```

**Minecraft version**<br />
`1.20.5+`

**Fields**<br />
`value` sets the durability value and must be at least `1`.<br />
`damage_block_break` and `damage_entity_hit` are booleans and default to `true`.

<Callout type="info">
  On modern versions, this replaces the legacy `durability` mechanic for item durability configuration.
</Callout>
</Details>

<Details>
<summary id="food"><strong>🍖 `food`</strong></summary>
Makes the item behave like food and defines how much hunger and saturation it restores.<br />

```yaml
myitem:
  components:
    food:
      nutrition: 6
      saturation: 7.2
      can_always_eat: false
```

**Minecraft version**<br />
`1.20.5+`

**Fields**<br />
`nutrition` sets the hunger restored and must be at least `0`.<br />
`saturation` sets the saturation restored and must be at least `0`.<br />
`can_always_eat` allows the item to be consumed even when the player is full and defaults to `false`.

<Callout type="info">
  On `1.20.5+`, prefer this component over the legacy `food` mechanic.
</Callout>
</Details>

<Details>
<summary id="consumable"><strong>🥤 `consumable`</strong></summary>
Controls how the item is consumed, including timing, animation, particles, and sound.<br />

```yaml
myitem:
  components:
    consumable:
      consume_seconds: 1.6
      animation: "eat"
      sound: "entity.generic.eat"
      has_consume_particles: true
```

**Minecraft version**<br />
`1.21.2+`

**Fields**<br />
`consume_seconds` is the consumption time in seconds and defaults to `1.6`.<br />
`animation` defines the use animation.<br />
`sound` defines the consume sound.<br />
`has_consume_particles` controls whether consume particles are shown and defaults to `true`.

<Callout type="info">
  Valid sound keys use Minecraft sound resource locations, such as `entity.generic.eat` or `item.honey_bottle.drink`.
</Callout>
</Details>

<Details>
<summary id="equippable"><strong>🛡️ `equippable`</strong></summary>
Makes the item equippable and defines slot-related behavior.<br />

```yaml
myitem:
  components:
    equippable:
      slot: HEAD
      model: "oraxen:my_helmet"
      equip_sound: "item.armor.equip_diamond"
      dispensable: true
      swappable: true
      damage_on_hurt: true
      camera_overlay: "oraxen:textures/misc/pumpkinblur.png"
```

**Minecraft version**<br />
`1.21.2+`

**Fields**<br />
`slot` is required and defines the equipment slot.<br />
`model` defines the model resource location.<br />
`equip_sound` defines the equip sound.<br />
`dispensable`, `swappable`, and `damage_on_hurt` default to `true`.<br />
`camera_overlay` defines the overlay texture.

**Slot groups**<br />
- `HAND`
- `OFF_HAND`
- `FEET`
- `LEGS`
- `CHEST`
- `HEAD`
- `BODY`
- `SADDLE`

<Callout type="info">
  On `1.21.2+`, this replaces older behaviors such as the legacy `hat` mechanic for wearable items.
</Callout>
</Details>

<Details>
<summary id="tool"><strong>⛏️ `tool`</strong></summary>
Defines tool mining behavior, including default speed and optional mining rules.<br />

```yaml
myitem:
  components:
    tool:
      damage_per_block: 1
      default_mining_speed: 4.0
      rules:
        - blocks: "minecraft:mineable/pickaxe"
          speed: 8.0
          correct_for_drops: true
```

**Minecraft version**<br />
`1.20.5+`

**Fields**<br />
`damage_per_block` sets the durability damage dealt per block and defaults to `1`.<br />
`default_mining_speed` sets the mining speed and defaults to `1.0`.<br />
`rules` defines optional mining rules for specific block tags or blocks.
</Details>

<Details>
<summary id="use_cooldown"><strong>⏱️ `use_cooldown`</strong></summary>
Adds a cooldown after using the item.<br />

```yaml
myitem:
  components:
    use_cooldown:
      seconds: 2.5
      group: "magic_wands"
```

**Minecraft version**<br />
`1.21.2+`

**Fields**<br />
`seconds` sets the cooldown duration in seconds and must be at least `0`.<br />
`group` defines the shared cooldown group.
</Details>

<Details>
<summary id="use_remainder"><strong>🪣 `use_remainder`</strong></summary>
Defines which item remains after the item is consumed or used up.<br />

```yaml
myitem:
  components:
    use_remainder:
      minecraft_type: "BOWL"
      amount: 1
```

```yaml
myitem:
  components:
    use_remainder:
      oraxen_item: "empty_bottle"
      amount: 1
```

**Minecraft version**<br />
`1.21.2+`

**Fields**<br />
`oraxen_item`, `minecraft_type`, `crucible_item`, `mmoitems_id`, `mmoitems_type`, and `ecoitem_id` define the item that is returned.<br />
`amount` sets how many of that item remain and defaults to `1`.
</Details>

<Details>
<summary id="jukebox_playable"><strong>🎵 `jukebox_playable`</strong></summary>
Makes the item playable in jukeboxes.<br />

```yaml
myitem:
  components:
    jukebox_playable:
      show_in_tooltip: true
      song_key: "oraxen:my_song"
```

**Minecraft version**<br />
`1.21+`

**Fields**<br />
`show_in_tooltip` controls whether the song appears in the tooltip and defaults to `true`.<br />
`song_key` is the namespaced song key.

**Format**<br />
`song_key` should be a namespaced key such as `minecraft:cat` or `oraxen:my_song`.

<Callout type="info">
  On `1.21+`, prefer this component over the deprecated legacy `music_disc` mechanic.
</Callout>
</Details>

<Details>
<summary id="tooltip_style"><strong>💬 `tooltip_style`</strong></summary>
Sets a custom tooltip style resource location.<br />

```yaml
myitem:
  components:
    tooltip_style: "oraxen:epic_tooltip"
```

**Minecraft version**<br />
`1.21.2+`
</Details>

<Details>
<summary id="item_model"><strong>🖼️ `item_model`</strong></summary>
Sets a custom item model resource location through the component system.<br />

```yaml
myitem:
  components:
    item_model: "oraxen:weapons/flame_blade"
```

**Minecraft version**<br />
`1.21.2+`
</Details>

<Details>
<summary id="generic"><strong>➕ `generic`</strong></summary>
Allows you to use any minecraft component, even if they are not specifically handled by Oraxen.<br />

```yaml
myitem:
  components:
    repairable:
      items:
        - minecraft:diamond
```

```yaml
myitem:
  components:
    custom_component:
      some_property: value
      another_property: 123
```

**Minecraft version**<br />
`1.21.3+`
</Details>



## Example

```yaml
myitem:
  itemname: "<gold>Traveler's Meal"
  material: BREAD
  components:
    max_stack_size: 16
    fire_resistant: false
    food:
      nutrition: 8
      saturation: 9.6
      can_always_eat: true
    consumable:
      consume_seconds: 1.6
      animation: "eat"
      sound: "entity.generic.eat"
      has_consume_particles: true
    use_remainder:
      minecraft_type: "BOWL"
      amount: 1
    use_cooldown:
      seconds: 1.0
      group: "food_items"
    tooltip_style: "oraxen:food/meal"
    item_model: "oraxen:food/travelers_meal"
```
